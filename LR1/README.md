# Лабораторная работа №1

## Курс: Киберфизические системы

### Тема: Обнаружение и распознавание объектов с использованием Ultralytics YOLOv11

## Описание работы

В рамках лабораторной работы исследуется применение моделей семейства **Ultralytics YOLOv11** к задаче **детекции дыма и огня** на изображениях с камер наблюдения. Задача прикладная — раннее обнаружение возгораний для систем промышленного и экологического мониторинга.

В качестве набора данных использован **D-Fire** — открытый датасет для детекции огня и дыма, уже размеченный в формате YOLO. Набор содержит два класса объектов:

- `smoke` (class id = 0)
- `fire`  (class id = 1)

В работе последовательно реализованы три этапа:

1. **Baseline** — обучение базовой модели `YOLO11n` с дефолтными параметрами.
2. **Улучшенный baseline** — проверка гипотез о улучшении: переход на более крупную модель `YOLO11s`, усиленные аугментации (mosaic + mixup + copy_paste), оптимизатор AdamW с cosine learning rate.
3. **Собственная модель** — своя YOLO-архитектура через кастомный YAML-конфиг, обученная с нуля; затем — та же модель с техниками из улучшенного baseline.

## Используемый датасет

**Название:** Smoke-Fire-Detection-YOLO (D-Fire)
**Источник:** Kaggle — [sayedgamal99/smoke-fire-detection-yolo](https://www.kaggle.com/datasets/sayedgamal99/smoke-fire-detection-yolo)
**Формат:** YOLO (папки `train/`, `val/`, `test/` с подпапками `images/` и `labels/`)

## Метрики качества

- **Precision** — доля истинно-положительных детекций; критична для систем пожарной сигнализации (минимизация ложных тревог).
- **Recall** — доля найденных возгораний; критична для безопасности (минимизация пропусков).
- **mAP@0.5** — средняя точность при IoU = 0.5, общая «обзорная» метрика качества.
- **mAP@0.5:0.95** — усреднение mAP по порогам IoU 0.5–0.95, показатель точности локализации рамок.

## Установка и запуск

### Вариант 1. Google Colab (рекомендуется)

1. Открыть `LR1_CYBER.ipynb` в Colab.
2. Включить GPU: `Среда выполнения` → `Сменить среду выполнения` → `T4 GPU`.
3. Выполнить ячейки ноутбука сверху вниз.
4. При первом обращении к Kaggle раскомментировать и выполнить:
   ```python
   import kagglehub
   kagglehub.login()
   ```
   Понадобится `kaggle.json` (API-токен из профиля Kaggle).

### Вариант 2. Локальный запуск

1. Клонировать репозиторий:
   ```bash
   git clone <URL_этого_репозитория>
   cd <имя_папки>
   ```

2. Создать виртуальное окружение:
   ```bash
   python -m venv .venv
   source .venv/bin/activate         # Linux/macOS
   # .venv\Scripts\activate           # Windows
   ```

3. Установить зависимости:
   ```bash
   pip install -U ultralytics kagglehub pandas pyyaml matplotlib opencv-python notebook
   ```

4. Настроить Kaggle API: положить `kaggle.json` в `~/.kaggle/` (Linux/macOS) или `%USERPROFILE%\.kaggle\` (Windows).

5. Запустить Jupyter:
   ```bash
   jupyter notebook
   ```
   Открыть `LR1_CYBER.ipynb` и выполнить ячейки по порядку.

## Выводы

1. **YOLO11n** — адекватный baseline. Запускается быстро, даёт понятную точку отсчёта.
2. Сравнение пунктов 2 и 3 показывает, что основной прирост качества даёт комбинация **более крупная модель + AdamW + cosine LR + расширенные аугментации**.
3. Собственная облегчённая YOLO-архитектура из пункта 4 уступает `YOLO11n` прежде всего из-за обучения *с нуля*, без предобученных весов COCO.
4. Применение техник улучшения ко собственной модели даёт прирост, но не позволяет догнать `YOLO11s improved` — подтверждает, что главный вклад в качество детекции даёт пара «хорошая архитектура + предобучение», а не просто расширение обучения.
5. Для прикладной задачи пожарного мониторинга рекомендуется **YOLO11s improved** как оптимальный баланс между качеством и скоростью инференса.

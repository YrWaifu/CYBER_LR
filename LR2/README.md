# Лабораторная работа №2

## Курс: Киберфизические системы

### Тема: Инференс LLM через локальный HTTP API сервера Ollama

## Описание работы

В рамках работы реализован минимальный клиент, обращающийся к локально запущенному серверу **Ollama** по HTTP для инференса языковой модели **Qwen2.5:0.5B**.

Python-скрипт читает набор из 10 произвольных запросов, последовательно отправляет их на сервер Ollama через эндпоинт `/api/generate`, собирает ответы модели и сохраняет итог в CSV-отчёт с двумя колонками: `prompt` и `response`.

## Что выполнено

- установлен `ollama`;
- загружена модель `qwen2.5:0.5b`;
- поднят локальный сервер Ollama, слушающий `http://127.0.0.1:11434`;
- написан Python-скрипт `client.py` с HTTP-клиентом к Ollama;
- выполнен прогон на 10 произвольных запросах;
- сформирован отчёт инференса `inference_report.csv`;
- все функции снабжены docstring-документацией.

## Используемый стек

- **Ollama** — локальный сервер для запуска LLM
- **Qwen2.5:0.5B** — компактная языковая модель (~500M параметров)
- **Python 3.10+** и библиотека **requests** для HTTP-запросов

## Структура репозитория

```
.
```

## Требования

- Linux / WSL2 / macOS (Ollama поддерживает и Windows native, но WSL стабильнее)
- Python 3.10+
- Доступ к интернету для первоначальной загрузки модели (`ollama pull`)
- ~1 ГБ свободного места под веса `qwen2.5:0.5b`

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone <URL_этого_репозитория>
cd <имя_папки>
```

### 2. Установка Ollama

Linux / WSL:

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

macOS: скачать установщик с [ollama.com/download](https://ollama.com/download).

Проверить установку:

```bash
ollama --version
```

### 3. Загрузка модели Qwen2.5:0.5B

```bash
ollama pull qwen2.5:0.5b
```

Команда скачает ~400 МБ весов и квантованных параметров модели.

### 4. Запуск сервера Ollama

В отдельном терминале:

```bash
ollama serve
```

По умолчанию сервер слушает `http://127.0.0.1:11434`.

Быстрая проверка доступности:

```bash
curl http://127.0.0.1:11434
# ожидаемый ответ: Ollama is running
```

### 5. Установка Python-окружения

```bash
python3 -m venv .venv
source .venv/bin/activate          # для Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 6. Запуск скрипта инференса

```bash
python client.py
```

Ход выполнения отображается в консоли: для каждого запроса выводится сам prompt, время ответа в секундах и первые ~120 символов ответа модели. После прогона файл `inference_report.csv` будет **перезаписан** полученным отчётом.

## Отчёт инференса

Файл `inference_report.csv` содержит две колонки:

| Колонка    | Описание                             |
|------------|--------------------------------------|
| `prompt`   | текст запроса, отправленного в LLM   |
| `response` | полный ответ модели                  |

В качестве запросов использованы 10 тем из разных областей (техническое объяснение, генерация кода, перевод, стихотворение, сравнение архитектур), подобранных так, чтобы нагрузить модель разнообразным вводом.

Пример вывода из отчёта (первая строка):

> **prompt:** Расскажи кратко, чем отличается машинное обучение от классического программирования.
>
> **response:** В классическом программировании разработчик явно пишет правила, по которым программа обрабатывает данные. В машинном обучении правила выводятся автоматически — модель обучается на примерах и сама находит закономерности в данных.
